# 变更日志

## 2026-06-11 | feat | 引入 os 维度并扩展 Linux 终端支持

- 变更内容：
  - 数据模型新增 `os: windows | linux | cross`：8 个模板加 `os` 字段，`tools/wabk_common.py` 新增 `infer_os`，SQLite（documents 表 + 索引）、Neo4j、JSONL 与 API（`/search?os=`、`/stats.by_os`）均输出 os。
  - 现有约 51 个 Windows 画像回填 `os: windows`。
  - 新增 Linux 垂直样例：nginx 应用 / 服务（systemd）/ 进程、systemd、systemd Service、Cron Job、`12_Linux持久化与配置/`（systemd unit、Crontab）及跨平台基线 [[服务持久化机制对比]]（`os: cross`）。
  - 项目更名：Windows应用安全基线画像库 → 终端应用安全基线画像库（含两个总览/规范文件重命名）。
  - 补齐缺失索引（启动方式 / 文件与数据 / 网络行为 / Linux持久化与配置），证据模板归位 `99_模板/`，新增启动方式模板与 Linux 持久化模板。
- 影响页面：README、AGENTS.override、CLAUDE.project、AI 维护规范、wiki-maintenance skill、全部模板与索引、`tools/*`、新增 Linux 画像。
- 依据来源：配置适配建议与目录结构 review 结论；nginx / systemd / cron 为通用 Linux 常识。
- 待验证问题：Linux 画像需补充真实终端观测；`config_persistence` 类型随 Linux 内容增长再评估是否拆分。

# 项目开发技能

当任务涉及代码编写、代码评审、调试、重构、配置调整、测试补充或构建修复时，使用本技能。

## 开始前

- 阅读 `CLAUDE.md`、`CLAUDE.project.md` 和 `.claude/rules/`。
- 确认相关代码、配置、测试和文档。
- 检查待修改区域是否命中架构阈值、生命周期状态分散、schema 双轨、复制风险或旧兼容层残留。
- 如涉及 AI / 自动化输出契约，区分 backend-owned、agent-owned 和 boundary-owned 字段。
- 如存在多个合理解释，列出解释和取舍，不静默选择。

## 写文件前

- 输出计划，列出涉及文件、逻辑修改点、验证方式和成功标准。
- 说明 Docker 卷挂载路径；不涉及时写“不涉及”。
- 说明项目覆盖层声明的治理硬门；没有配置时写明替代验证和缺口。
- 如触发替换旧设计模式，列出删除、迁移、保留清单。
- 如涉及用户可见功能，说明新增按钮、状态、后台任务、API 或 schema 字段对应的用户任务。

## 执行时

- 普通小改只修改完成目标必需的文件和逻辑。
- 优先复用健康的项目已有模式，不引入投机性抽象或依赖。
- 不把个人路径、token、私有账号配置或运行态数据写入团队文件。
- 发现无关问题时记录并说明，不顺手重构。

## 完成后

- 运行相关测试、构建、类型检查或人工验证。
- 对照原始目标说明是否达成。
- 说明未验证项、原因和残余风险。
- 如影响公开 API、配置/env、Docker 默认值、agent profile 或 workspace 路径，同步 README 或 docs。

# 生命周期基线条件规则抽取规范

## 1. 定位

本规范定义如何从 Markdown 画像库抽取可用于 STIX 行为事实图分析的生命周期条件规则。

生命周期规则用于判断同一个运行时进程实例是否同时满足：

```text
创建时基线：父进程、启动用户、路径、命令行、启动方式
运行时基线：子进程、网络、文件、注册表/配置、模块、持久化变化
证据完整性：判断所需证据是否齐全
```

## 2. 结构化规则块

规则块放在画像正文固定章节中，不放入 frontmatter。

推荐章节名：

````markdown
## 结构化生命周期基线

```yaml
lifecycle_baseline:
  version: 1
  applies_to: process
  phase: lifecycle
  creation:
    required_evidence:
      - parent_process
      - image_path
      - command_line
      - user
    expected_parent_processes:
      - services.exe
    expected_user_contexts:
      - local_system
    suspicious_image_locations:
      - user_writable_dir
      - temp_dir
      - downloads_dir
    command_line_contains_any:
      - EncodedCommand
  runtime:
    required_evidence:
      - child_processes
      - network_connections
      - file_activity
      - registry_activity
    suspicious_child_processes:
      - powershell.exe
      - cmd.exe
    risk_escalates_when:
      - network_after_spawn
      - file_drop_after_spawn
      - persistence_after_spawn
  false_positive:
    allowed_contexts:
      - authorized_service
      - approved_change
```
````

## 3. 字段语义

| 字段 | 说明 |
|---|---|
| `version` | 规则块版本，当前为 `1`。 |
| `applies_to` | 规则适用对象：`process`、`service`、`process_relation`、`process_behavior`、`security_baseline`。 |
| `phase` | `creation`、`runtime` 或 `lifecycle`。 |
| `creation.required_evidence` | 创建时判断必须具备的证据。 |
| `creation.expected_parent_processes` | 常见父进程名。 |
| `creation.expected_user_contexts` | 用户上下文类型，不写死企业真实账号。 |
| `creation.suspicious_image_locations` | 异常路径类别。 |
| `creation.command_line_contains_any` | 命中任一项即提高风险的命令行片段。 |
| `runtime.required_evidence` | 运行时判断必须具备的证据。 |
| `runtime.suspicious_child_processes` | 高关注子进程。 |
| `runtime.risk_escalates_when` | 运行时风险升级条件。 |
| `false_positive.allowed_contexts` | 可降级的业务上下文或误报条件。 |

## 4. 用户上下文类型

知识库只沉淀用户上下文类型，不沉淀企业真实账号：

```text
interactive_user
local_system
local_service
network_service
service_account
root
systemd_service_user
scheduled_task_user
unknown_or_unexpected
```

真实用户、SID、域账号、服务账号白名单、授权清单由 EDR、资产台账、IAM 或变更系统提供。

## 5. 派生规则

在页面尚未补齐结构化规则块时，转换工具可以从 `process_relation` 页面派生创建时规则：

```text
parent_process + child_process + relation + normality + risk_level
```

派生规则只说明该父子关系在 KB 中是否已知及其风险等级，不能单独证明生命周期安全。运行时证据仍需结合进程、网络、文件、注册表和安全基线规则继续判断。

## 6. Cypher 输出模型

转换工具生成以下派生节点：

```cypher
(:KbLifecycleRule {
  rule_id,
  rule_name,
  source_type,
  source_path,
  source_title,
  phase,
  applies_to,
  normality,
  risk_level,
  confidence,
  conditions_json,
  evidence_requirements
  parent_process_names,
  child_process_names,
  runtime_child_process_names,
  command_line_terms,
  image_location_terms,
  runtime_indicators,
  false_positive_contexts
})

(:KbDocument)-[:DEFINES_LIFECYCLE_RULE]->(:KbLifecycleRule)
```

STIX 行为事实图中的进程节点仍使用运行时图模型，不与 `KbLifecycleRule` 混写。

转换工具还会生成 `out/lifecycle_analysis_queries.cypher`，提供以 STIX `Process` 为锚点的参数化查询模板。查询模板不直接给出最终恶意判断，只输出创建时规则命中、运行时证据摘要和证据完整性骨架，最终 `safe / risky / deviation / baseline_gap / evidence_insufficient` 由分析服务按证据收敛。

## 7. 判断结论

生命周期判断服务应按以下结果收敛：

```text
safe
risky
deviation
baseline_gap
evidence_insufficient
```

缺少创建时证据或运行时证据时，不允许输出 `safe`。

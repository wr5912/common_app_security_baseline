# 数据库中间件服务常驻

## 1. 基线说明

数据库和中间件服务在服务器或开发机上常见，但在普通办公终端常驻需要结合资产角色判断。

## 2. 关联对象

- [[Microsoft SQL Server]]
- [[MySQL]]
- [[PostgreSQL]]
- [[MongoDB]]
- [[Redis for Windows]]

## 3. 合法条件

```text
资产角色为数据库服务器或开发环境
监听范围符合业务设计
服务账户权限最小化
配置和数据目录可追溯
```

## 4. 异常条件

```text
普通终端出现数据库服务
数据库端口对非授权网段开放
服务账户权限过高
数据目录或二进制路径异常
```

## 5. 推荐处置

优先核验资产角色、授权状态、服务 ImagePath、签名、启动账户、监听端口、网络目的地址和变更时间线。不要只凭服务名判断恶意。

## 6. 证据与来源

- 来源页面：[[Windows常见应用服务基线清单]]
- 待验证：企业资产角色、授权清单、真实命令行和网络观测。

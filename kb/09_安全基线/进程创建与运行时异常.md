# 进程创建与运行时异常

## 1. 基线说明

本基线用于统一刻画进程创建时和运行时的安全判断边界。它不维护具体 hash、证书指纹、信誉分或单版本白名单；这些动态情报由 EDR、资产台账、软件分发或情报系统提供。

## 3. 正常条件

```text
父进程、启动账户、完整路径、命令行参数、工作目录和启动方式能由应用/服务/启动方式画像解释
子进程、文件/注册表/配置访问、网络目的地址和监听端口符合对应画像
资产角色、授权清单、变更记录和用户交互上下文能够解释该进程活动
```

## 4. 异常条件

```text
从用户可写目录、临时目录、下载目录、网络共享或异常挂载路径启动
父子进程关系偏离画像，尤其是 Office、浏览器、PDF、远控或服务进程拉起脚本解释器/系统管理工具
命令行包含隐藏执行、绕过策略、内联脚本、下载执行、异常代理、异常配置目录或凭据/令牌相关参数
运行时发生异常外联、持久化写入、敏感文件访问、异常监听端口或未知子进程创建
```

## 5. 误报条件

```text
企业授权的软件分发、远程管理、监控采集、备份恢复、开发调试或自动化运维场景
完整命令行固定且可解释，路径/账户/父子关系与部署规范一致
网络目的地址、文件落点和配置路径均能由业务画像或变更记录解释
```

## 6. 证据需求

```text
进程创建事件：时间、主机、用户、进程 GUID/PID、父进程 GUID/PID、完整命令行、当前目录、启动账户、镜像路径
运行时证据：子进程、文件/注册表/配置访问、网络连接、监听端口、模块加载、服务/计划任务/Run Key/systemd/cron 变化
外部核验：签名、hash、证书链、信誉、首次出现时间、资产授权和变更单由专门系统提供
```

## 7. 推荐处置

优先按“创建链路是否可解释 -> 参数是否可解释 -> 运行时行为是否可解释 -> 资产/授权是否可解释”的顺序降噪或升级。无法解释时进入人工复核或检测规则候选池。

## 8. 回归测试建议

```text
新增或修改进程/父子关系画像后运行 audit_process_behavior_baseline.py
严格双链审计必须确认 [[进程创建与运行时异常]] 可解析
SQLite/Neo4j 重建后确认 sections 和 links 可消费
```

## 9. 结构化生命周期规则

```yaml
lifecycle_baseline:
  version: 1
  applies_to: process_behavior
  phase: lifecycle
  creation:
    required_evidence:
      - parent_process
      - image_path
      - command_line
      - user
      - start_method
    image_location_any:
      - user_writable_dir
      - temp_dir
      - downloads_dir
      - network_share
      - abnormal_mount
    command_line_contains_any:
      - -EncodedCommand
      - -ExecutionPolicy Bypass
      - -WindowStyle Hidden
      - IEX
      - DownloadString
      - FromBase64String
  runtime:
    required_evidence:
      - child_processes
      - network_connections
      - file_activity
      - registry_or_config_activity
      - module_loads
      - persistence_changes
    risk_escalates_when:
      - abnormal_external_connection
      - persistence_write
      - sensitive_file_access
      - abnormal_listening_port
      - unknown_child_process
  false_positive:
    allowed_contexts:
      - authorized_software_distribution
      - remote_management
      - monitoring_collection
      - backup_restore
      - development_debugging
      - approved_automation
```
